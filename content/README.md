# PloverCRM Website Content Workspace

This folder is for planning and drafting website content before page design begins.

Use these source-of-truth files when drafting pages:

- [../landing-page-content-strategy.md](../landing-page-content-strategy.md) — homepage and positioning strategy.
- [../landing-page-final-copy.md](../landing-page-final-copy.md) — approved homepage copy.
- [../design.md](../design.md) — visual system direction for later design work.
- [../landing-page-metric-research.md](../landing-page-metric-research.md) — research support for metric-led content.

## Content rules

- Keep PloverCRM WordPress-first.
- Sell lead control after capture, not generic CRM software.
- Keep pages direct, calm, specific, and conversion-focused.
- Create content first. Treat design notes as secondary until copy is approved.
- Prefer fewer, stronger educational comparison pages over many thin one-off competitor pages.
- Use the page template in each file before writing final copy.
